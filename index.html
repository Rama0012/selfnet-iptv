SelfStreaming

Simply put, Selfnet's SelfStreaming service allows you to watch TV on your computer.

In-Browser Stream List

Currently, all implemented methods deliver the same quality. Just choose the one that works best for you.

  • working:
    • Multicast RTP – recommended, most efficient in the Selfnet intranet
    • Unicast HTTP – if you are in a wireless network or if you can't receive multicast
  • might become available in the future:
    • HTTP Live Streaming (HLS)
    • RTMP
    • your suggestion?

SAP Stream List

This works in all players that support SAP session discovery (VLC or XBMC, please tell us if you know any others).

  1. Download and install VLC media player. If you already have VLC installed, make sure you are using the newest version.
  2. Activate the playlist view:
  3. Scroll down to "Local Network" and click on "Network streams (SAP)":

    You should see the stream list getting filled.
  4. If you are using VLC 2.1.0 or newer, you can see additional information by enabling the "Description" column in the stream list:
  5. When you double-click on a stream, it should start playing within a few seconds.

Troubleshooting

  • Wireless networks deal badly with multicast. Use the HTTP-tunneled streams instead. Even then, the streams might buffer very slowly, depending on your signal quality.
  • If you are using a router in your room, you probably have to configure two UDP port forwardings: one for the video streams on port 1234 and one for the stream announcements on port 9875. (This has nothing to do with port forwardings in the Selfnet NAT.)
  • If something used to work for you and now it doesn't, check our Twitter account @Selfnet_eV for recent announcements.
  • Our main website has some information about the SelfStreaming service, too.
